# debounce 防抖

防抖策略(debounce)是当事件被触发后，延迟n秒后再执行回调函数，如果在这n秒内事件被再次触发，则重新计时。

# 按需引入

import Debounce from 'easy-project-vue/dist/directives/debounce.js'
Vue.use(Debounce);

# 何时使用

使用debounce指令，会给dom绑定一个点击事件，在你设定的毫秒内（默认300毫秒）点击，只会触发1次事件。

  • 好处：它能够保证用户在频繁触发某些事件的时候，不会频繁的执行回调，只会被执行一次。

  • 例子：如果有人进电梯（触发事件），那电梯将在10秒钟后出发（执行事件监听器），这时如果又有人进电梯了（在10秒内再次触发该事件），我们又得等10秒再出发（重新计时）。

  • 场景：适用于用户可能会多次请求接口的场景，以节约请求资源。

    • 用户在输入框连续输入一串字符时，可以设置防抖策略，只在输入完后，才执行查询的请求。
    • 用户在点击搜索按钮时，可以设置防抖策略，解决用户狂点按钮、疯狂请求后台的问题。

# 配置项

参数 说明 类型 默认值
debounceDuration 设定在多少毫秒内点击，只会触发1次事件 number 300

注册配置

import Debounce from 'easy-project-vue/dist/directives/debounce.js'
let Options = { debounceDuration:1000 }

Vue.use(Debounce,Options);

# 用法示例

请你疯狂点击下面的按钮，在0.3秒内无论你点击多少次，防抖指令只会触发1次请求： 

<template>
    <div>
        <button v-debounce="debounceClick">发送请求</button>
        <div>请求后台次数：{{count}}</div>
    </div>
</template>
<script>

  export default {
        data() {
            return {
                count: 0
            }
        },
        methods: {
            debounceClick() {
                //请求后台接口...
                this.count++
                console.log('我被点击了')
                console.log('我在0.3只会被触发一次！')
            }
        }
  }
</script>
