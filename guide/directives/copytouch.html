# copytouch 复制框选

保证复制框选时不会触发点击事件

# 按需引入

import Copytouch from 'easy-project-vue/dist/directives/copytouch.js'
Vue.use(Copytouch);

# 何时使用

让我们来看这样一个场景，现在我们给下面这一大段文字绑定点击事件，点击下面的文字试试看：

<template>
       <p @click="click">
           我是一段文字，我是一段文字，<span style="color: red">请复制我</span>，我是一段文字，我是一段文字，我是一段文字，我是一段文字，我是一段文字，我是一段文字，我是一段文字，我是一段文字，我是一段文字，我是一段文字，我是一段文字，我是一段文字，我是一段文字，我是一段文字，我是一段文字，我是一段文字，我是一段文字，我是一段文字，我是一段文字，我是一段文字，我是一段文字，我是一段文字，我是一段文字，我是一段文字，我是一段文字，我是一段文字，我是一段文字，我是一段文字...
       </p>
   </template>
   <script>
       export default {
           name: "CopytouchDemo1",
           methods: {
               click() {
                   alert('我被点击了')
               }
           },
       }
   </script>

现在请你框选并复制这段标红文字："请复制我"，此时便会触发你设定好的点击事件。

这与我们的初衷相违背，因为用户在框选时并不想触发点击事件，这会给用户带来槽糕的操作体验。

于是，复制框选指令诞生了，用这个指令绑定点击事件后，可以保证复制框选时不会触发点击事件，具体使用可以看下面的例子。

# 用法示例

在使用v-copytouch指令绑定点击事件后，请你再次框选试试看： 

<template>
    <p v-copytouch="click">
      我是一段文字，我是一段文字，我是一段文字，我是一段文字，我是一段文字，我是一段文字，我是一段文字，我是一段文字，我是一段文字，我是一段文字，我是一段文字，我是一段文字，我是一段文字，我是一段文字，我是一段文字，我是一段文字，我是一段文字，我是一段文字，我是一段文字，我是一段文字，我是一段文字，我是一段文字，我是一段文字，我是一段文字，我是一段文字，我是一段文字，我是一段文字，我是一段文字，我是一段文字，我是一段文字...
    </p>
</template>
<script>
  export default {
       methods: {
          click() {
            alert('我被点击了')
            console.log('只有点击后才会触发点击事件！')
            console.log('框选时不会触发点击事件！')
          }
        }
  }
</script>
