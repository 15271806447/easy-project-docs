# 二次封装的需求场景

二次封装，是指因为业务上的需要，对element-ui、antd、e-chart等其他组件库，做二次封装。以下是常见的二次封装场景：

# 1.封装UI样式

说个我做项目遇到的情况，项目里用到组件库的是antd，antd的搜索框长这样：

但是ui图长这样：

如果要实现和ui图一样的效果，你就需要利用前缀和后缀的插槽，加上搜索图标和清空图标：

<a-input :placeholder="placeholder" :style="inputStyle" @keydown.enter=handleEnter v-model="searchValue"

有时还需要改改样式：

那么这样就带来一个问题，要知道搜索框可是复用频率很高的，如果每次都需要粘贴同样的插槽和样式代码，那不是很憨憨吗？

而且后期维护，如果要改搜索框的样式，那就得改好几处地方，所以这一段组件是一定要封装的。

# 二次封装的原则和方法

WARNING

二次封装时，必须暴露原组件的所有props、事件、插槽和方法

如果做二次封装，没有暴露组件库本身的属性和方法。那说实话，你这不叫封装组件，你这叫“坐牢”，只有封没有装那能叫封装吗？

还是以封装antd的输入框为例，我只是对样式做了一些更改，那么antd输入框中的props、事件、插槽和方法，是应该暴露出来给父组件的，否则封装的组件压根就没法用。

那我这里怎么绑定呢？不可能一个个照着文档把props全绑上去吧？

<a-input placeholder="placeholder" :defaultValue="defaultValue"
         :disabled="disabled" @change="onChange" :maxLength="maxLength" />

//憨憨的做法：照着文档一个个地绑定props
props:{
    placeholder: {
      type: String,
      default: ''
    },
    defaultValue: {
      type: String,
      default: ''
    },
    //把所有的props一个个填上去......
}

真当程序员是码农啊？别干这些“重复”的体力活了，具体怎么做呢？这里讲一下封装技巧：

# 1）绑定$attrs暴露props

实现思路：用计算属性获取this.$attrs的值，然后用v-bind绑定，下面antd的弹窗组件为例： HTML

<a-modal v-bind="_attrs">

将一些未处理的参数或特殊处理的参数绑定到 a-modal 上

computed: {
    _attrs() {
      let attrs = { ...this.$attrs }
      return attrs
    },
}

注意：用$attr绑定，会丢失掉函数。比如antd表格组件的columns的customRender。

父组件：

data() {
      return {
        description: '审计模型任务结果管理页面',
        // 表头
        columns: [
          {
            title: '#',
            dataIndex: '',
            key: 'rowIndex',
            width: 60,
            align: 'center',
            //这里customRender无法通过attr传给子组件
            customRender: function(t, r, index) {
              return parseInt(index) + 1
            }
          },
    }

这种情况就只能手动穿透了：

<template>
 <a-table v-show="dataSource.length!==0&&!loading" :columns="columns" :pagination="false" :dataSource="dataSource"
             :scroll="scroll"
             v-bind="_attrs"
             v-on="$listeners">
</template>
 export default {
    data() {
      return {
        // 内部已经使用的 slots、attrs ，不再处理
        usedSlots: ['action'],
        usedAttrs: ['pagination', 'columns']
      }
    },
    props: {
      //这里必须要声明为colums，来手动传参。
      //否则会用$attrs传值，customRender无法渲染成函数
      columns: {
        type: Array,
        default: () => []
      },
    },
    computed: {
      _attrs() {
        let result = JSON.parse(JSON.stringify(this.$attrs))
        this.usedAttrs.forEach(item => {
          delete result[item]
        })
        return result
      },
    }
}

主要原因是，vue内部处理数据传参的时候，传的时候会转成字符串。

类似于这样处理：

JSON.parse(JSON.stringify(data))

vue这里做了深度克隆，是为了避免传内存地址，导致数据这边改了，由于共用内存地址，界面的其他地方也改动了，导致数据维护的异常。

这是为了保护单向数据流，最简单的做法。

# v-bind的参数不会覆盖自己手动绑的attr

这里再讲一个注意点

<template>
 <a-table :pagination="false" v-bind="_attrs">
</template>
 export default {
   data() {
      return {
        // 内部已经使用的 slots、attrs ，不再处理
        usedSlots: ['action'],
        //这里没必要做这个usedAttrs，v-bind的优先级没有手动绑定的高
        usedAttrs: ['pagination', 'columns','dataSource']
      }
    },
    computed: {
      _attrs() {
      //这里我做了一次已用attr的过滤，其实是没必要的，他还是会以我绑定pagination的值（false）为准
      //哪怕我外面正常传了pagination的值，界面还是不会显示分页的，因为这里v-bind的优先级没有手动绑定的高。
        let result = JSON.parse(JSON.stringify(this.$attrs))
        this.usedAttrs.forEach(item => {
          delete result[item]
        })
        return result
      },
    }

# 2）绑定$listeners暴露事件

<a-modal
    @ok="handleOk"
    @cancel="handleCancel"
  >

其实这里我们需要抛change等事件给父组件，如果一个一个事件的写emit太憨了，有十个emit难道手动写十次emit吗？

下面推荐一个简单的方法：v-on="$listeners"

这段代码可以动态捕捉到emit事件

<a-modal
    v-on="$listeners"
  >

这样就不用手动再抛事件，给a-modal的组件了。 但是在开发中，我们有时需要覆写组件的@change等方法，那么这里需要就注意一个问题：

v-on=“$listeners“与重写事件有冲突，会执行两次。

说直白点就是，如果你手动写了v-on="$listeners"，然后你又自己重写了其中的事件，那么这个事件就会执行两次。

比如 element 组件 el-select，change 事件重写，返回更多的参数：

<template>
 <el-select @change="changeSelect" v-on="$listeners">xxx</el-select>
</template>

 export default {
    method: {
        changeSelect(val) {
            this.$emit('change', val) 
        }
    }
}

使用：

<EccSelect @change="selectChange"></EccSelect>

结果： 执行2次 , 多一个 undefined 第一个是v-on="$listeners"，第二个是@change="onSelectChange

解决： 去掉 @change=“changeSelect”， 使用 new$listeners 继承原有属性，并覆盖原有事件

<template>
<!--错误的写法，这里就不需要再绑定on事件-->
<el-select v-on="new$listeners" @change="changeSelect">xxx</el-select>
<!--正确的写法，绑定newnew$listeners即可-->
<el-select v-on="new$listeners">xxx</el-select>
</template>
 export default {
    computed: {
        new$listeners() {
          return Object.assign(this.$listeners, {
           //在这里覆盖原有的change事件
            change: this.changeSelect
          })
        }
    },
    method: {
      changeSelect(val) {
           //你自己的业务逻辑
           //...
            this.$emit('change', val) 
      }
    }
}

# 3）绑定$slots和$scopedSlots暴露插槽

同样的道理，插槽也是必须要暴露给父组件的，主要靠Vue提供的$slots和$scopedSlots，具体方法如下：

  <a-modal>
  <!--这里可以写自定义的slot-->
    <template slot="title">
      <slot name="title">我是自定义的{{ title }}</slot>
    </template>
    <slot></slot>
      <!-- 处理 scopedSlots -->
      <!-- 2.6新写法：<template #[slotName]="slotProps" v-for="slotName in scopedSlotsKeys">-->
      <template :slot="slotName" slot-scope="slotProps" v-for="slotName in scopedSlotsKeys">
        <slot :name="slotName" v-bind="slotProps"></slot>
      </template>

      <!-- 处理 slots -->
      <!-- 2.6新写法：<template #[slotName] v-for="slotName in slotsKeys">-->
      <template :slot="slotName" v-for="slotName in slotsKeys">
        <slot :name="slotName"></slot>
      </template>
  </a-modal>

计算属性拿到父组件注册的插槽：

export default {
   data() {
      return {
        // 内部已经使用的 slots ，不再处理
        usedSlots: ['title'],
      }
   },
   computed: {
    slotsKeys() {
      return Object.keys(this.$slots).filter((key) => !this.usedSlots.includes(key))
    },
    scopedSlotsKeys() {
      return Object.keys(this.$scopedSlots).filter((key) => !this.usedSlots.includes(key))
    },
  },
}

再举个ant表格的例子：

<template>
    <a-table :columns="columns" :pagination="false" :dataSource="dataSource"
               :scroll="scroll"
               v-bind="$attrs"
               v-on="_listeners">
        <template :slot="slotName" slot-scope="text, record, index" v-for="slotName in scopedSlotsKeys">
          <slot :name="slotName" v-bind="{text, record, index}"></slot>
        </template>

        <!-- 处理 slots -->
        <template :slot="slotName" v-for="slotName in slotsKeys">
          <slot :name="slotName"></slot>
        </template>
      </a-table>
 <template>
 export default {
   data() {
      return {
        // 内部已经使用的 slots ，不再处理
        usedSlots: ['title'],
      }
   }
   computed: {
    slotsKeys() {
      return Object.keys(this.$slots).filter((key) => !this.usedSlots.includes(key))
    },
    scopedSlotsKeys() {
      return Object.keys(this.$scopedSlots).filter((key) => !this.usedSlots.includes(key))
    },
  },
}

注意这里，父组件使用的时候需要用到es6对象结构，跟antd的默认用法不一样，antd是直接用的rander方法处理插槽的参数，所以可以传多个参数，可以这样写：

<template :slot="slotName" slot-scope="text, record, index" v-for="slotName in scopedSlotsKeys">
</template>

# 4）暴露实例的方法

拿element-ui的 「Cascader级联选择器」 来说吧，我有一个需求，需要用到getCheckedNodes()这个方法

这里是我的父组件要调用这个方法，如果手动的注册事件，一层层的传递也挺麻烦。

//二次封装组件
 export default {
    methods:{
        getCheckedNodes(){
            return this.$refs.cascader.getCheckedNodes()
        },
        toggleDropDownVisible(visible){
            return this.$refs.cascader.toggleDropDownVisible(visible)
        }
        //...
    }
}

具体调用方法如下，二次封装的组件暴露出实例，利用实例再调用element内部的方法：

 export default {
//二次封装组件
methods:{
    getCascader(){
        return this.$refs.cascader
       }
}
}

父组件要调用实例方法，可以先获取实例，再调用方法（即链式调用）：

 export default {
//引入上面封装组件的父组件
methods:{
    loadData() {
        this.$nextTick(() => {
            this.$refs.child.getCascader().getCheckedNodes()
            this.$refs.child.getCascader().toggleDropDownVisible(true)
            //只要拿到这个实例，this.$refs.child.getCascader()，想调方法后面跟链式调用即可
        })
    }
}
}
